# Incident Response Starter Playbook

This playbook defines the minimum viable incident response capability
for organizations supported by Bonfim Cloud Tech.

Incident response is treated as an operational discipline,
not an emergency improvisation.

## Objectives
- Detect security incidents early
- Contain impact and limit blast radius
- Preserve evidence for analysis and audit
- Restore services safely
- Learn and improve after each incident

## Incident Definition
An incident is any event that:
- Compromises confidentiality, integrity or availability
- Violates security or governance policies
- Exposes data or credentials
- Indicates malicious or unauthorized activity

## Incident Categories
- Identity compromise (credentials, privilege abuse)
- Data exposure or leakage
- Malware or ransomware
- Service disruption or denial of service
- Misconfiguration with security impact
