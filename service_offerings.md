# Service Offerings & Engagement Model

Bonfim Cloud Tech delivers architecture-led security and governance.
Services are designed to be modular, auditable and outcome-driven.

## Service Principles
- Architecture before tools
- Identity-first security
- Governance embedded, not optional
- Documentation as deliverable
- Vendor-aware, not vendor-locked

## Core Service Lines

### 1. Cloud Security Baseline (Entry)
Purpose:
Establish a secure minimum baseline for organizations already using cloud.

Includes:
- IAM baseline (least privilege)
- Storage exposure review
- Network segmentation review
- Logging and monitoring enablement
- Risk summary and remediation plan

Outcome:
- Reduced attack surface
- Immediate risk visibility
- Actionable remediation steps

---

### 2. Governance & Compliance Foundation
Purpose:
Embed governance and LGPD-ready controls into cloud operations.

Includes:
- Governance model definition
- Policy framework (access, data, logging)
- Role and responsibility mapping
- Evidence and audit readiness
- Risk register initialization

Outcome:
- Clear accountability
- Compliance-aligned operations
- Audit-ready documentation

---

### 3. Zero Trust Architecture Design
Purpose:
Design a modern Zero Trust architecture aligned with business needs.

Includes:
- Identity-centric access model
- Application-level protection strategy
- Network microsegmentation design
- Data-centric protection model
- Security telemetry and monitoring plan

Outcome:
- Reduced lateral movement
- Strong identity enforcement
- Resilient security posture

---

### 4. Incident Response Readiness
Purpose:
Prepare organizations to respond to security incidents effectively.

Includes:
- Incident response playbook
- Roles and escalation paths
- Logging and evidence strategy
- Tabletop exercise or simulation
- Post-incident review framework

Outcome:
- Faster detection and response
- Reduced operational chaos
- Regulatory and legal preparedness

---

## Engagement Model

### Assessment-Based Engagement
- Fixed scope
- Time-bound
- Documentation-focused
- Clear deliverables

### Architecture-Led Projects
- Defined milestones
- Iterative reviews
- Architecture and governance artifacts
- Knowledge transfer included

### Advisory Retainer (Later Stage)
- Periodic reviews
- Architecture and governance oversight
- Security posture evolution
- No operational dependency created

## Delivery Artifacts
Each engagement produces:
- Architecture diagrams
- Security baselines
- Governance documentation
- Risk and remediation reports
- Implementation guidance

Artifacts remain with the client.

## What We Do Not Do
- No tool reselling
- No black-box security
- No undocumented changes
- No permanent client dependency

## Value Proposition
Bonfim Cloud Tech delivers:
- Reduced cyber risk
- Structural security improvements
- Governance by design
- Long-term operational resilience

Security is delivered as architecture, not as fear.
